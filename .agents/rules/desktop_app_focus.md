trigger: product_scope_or_feature_planning
description: Rule for keeping repository decisions aligned to the desktop-first product focus
---

- Treat the desktop app as the primary product and source of truth for feature completeness.
- Treat the online/browser build as a demo with intentionally reduced functionality unless the user explicitly requests demo-specific work.
- When planning or implementing changes, optimize UX, flows, and technical decisions for the desktop app first.
- Do not delay or dilute desktop app improvements just to preserve feature parity with the online demo.
- If a change affects both surfaces, document intentional demo limitations or deferred parity work in the relevant plan or docs update.
